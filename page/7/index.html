日记

今天离职了，退了很多群，微信删了很多未曾谋面的网友，手机上也不会出现哪些滴滴哒哒烦人的声音了，整个人都感觉轻松了不少，心情也变得舒畅，从去年入职到今天离职，感觉自己都崩着个一个非常紧的弦，愈拉愈紧，简直喘不过气来，也许是因为我很久都没有接触过社会，想要做好，又担心做不好。这个适应的过程真不好受，但是感觉自己如果第二次进其他公司的话，情况应该会好很多吧。 跳出这个围城，发现自己的判断也会变得理性客观一点，也许，还是自己这样的经历太少了。 得总结。 1.工作的时候要找机会锻炼自己的基本功，了解公司技术栈的每一端，不说精通，了解是必要的。因为每个人负责的模块可能只是整体当中很小的一部分，但是为了你的技能发展，需要利用空余时间将项目的整个架构，模块作用弄清楚，初级-> 高级 -> 架构师 -> 全栈 2.周末的时间一定不要浪费，优秀的人和一般的人的差别在于业余时间的利用，周末来临之前，一定得给自己制定个计划，周计划要与月计划统一，月计划要和年计划统一，不要做一个无头苍蝇。回想这一年的周末时光，实在是太浪费了。 3.没有被生活打垮的人和一般的人有个显著的区别，就是前者永远都能有热情去迎接生活中的各种事和人，依然有热爱生活的初心，给自己一句话，不要被一时的挫折所羁绊，未来的机会还是会留给有心之人。 4.没事的时候多看看各大招聘网站啊。

回望

心情烦闷，晚上骑着单车四处走了走。 讲真，上海是一个非常美丽的城市，不同于其他大城市，个人觉得上海很多地方的美景都可聘美于很多其他城市的地标型建筑，尤其是上海的夜景。 晚上，我从梅川路骑车出发，一路向东，沿着金沙江，途径旁边的居民区，学校，大型商场，写字楼，还有各种各样的便利店，路上的一切一切都似乎是在向刚来上海的人炫耀着这是一个非常摩登的大都市。骑到十字路口的时候，我都忍不住想停下一会儿，看一下周围的繁华 ，观察着路上这群行走匆匆的人们，有时候我也会突发怪想，他们大晚上出来是干啥的呢，仿佛他们都有着一个明确的要去的地方。他们应该来自祖国各地，有湖南的，有东北的，就像我们公司的人员组成一样，他们或许可以在老家选择一个稍微悠闲点儿的生活，但现在，他们是这个魔都的一份子。不知为何，对于繁华高楼耸立的都市里，无论是在武汉，还是上海，我都会有一种迷恋，似乎是站在车来车往的行人中间，自己也不再孤独。城市的便利又可以让自己专注于庞大社会机器上留给自己的那个扭螺丝的部分，多元的选择又可以让我们的业余变得有趣，每想到此，都会让我坚定一个念头，努力挣钱，争取在大城市上生存下来。 幼时念书的时候，除开学校，几乎很少到外面走一走，童年的很多时光我是趴在家里的楼顶上，望着远处的大山发呆，犹然记得第一次到比较远的地方时，还是在升高中入学考试的时候，那时我14岁，第一次到武汉，晚上住酒店，兴奋的睡不着，11点的时候还能看到外面灯火通明时，内心很受震撼，在我们的那个小山村里，每天晚上七八点，人们就会关灯睡觉，外面也是一片漆黑，虽然说现在教育已经很普及，但是在我生长的这个城镇里，还是有大部分的人会世居在这里，出生，死亡，都在这块土地上完成，慢节奏的生活固然可以享受的到慢的幸福，但终究还是会被同时代其他快速发展的东西淘汰，出于达尔文的物竞天择原理，我也不会选择那些慢节奏的职业，至少40岁之前不是。 [playlist type=”video” ids=”62”] 未来充满机遇，自己还需历练。

回望之旅

今天9月1号，又到了学生开学的日子，毕业至今，已经有一年多的时间了，一年当中，感触挺多。 以前当学生的时候，总是会对下一个阶段充满期待，在初进新环境时，自己总是能有极大的热情，认为对未来还抱有幻想，初升高的时候，期待更好的生活环境与更nice的老师同学，高升大的时候，期待着更大的选择空间和广阔的学习天地，大学进社会的时候，又期待着更多的独立和财务自由，而当我发现在处于终点站的社会的时候，我已经没有了期待，每天的重复性工作，让我有点不适应，这种岁月的增加除了只会让自己向熟练工靠近一点外，真的感觉别无他用，生活令我失望，我也明白了一个道理，外面的世界终究会有个终点，如果将人生的动力寄托在发现外面的世界或者有意思的人，这种初衷本身就是愚蠢的，因为你终究会发现你的动力来源是会枯竭的，此番经历后，我也有了一个感触，外在世界已经对我不能产生比较大的诱惑力了，至少现在，我的内心还算比较平静，因为我有了新的发现。 我本身是一个偏内向的人，独处使我更自在，舒适，当从学校到社会中一个大的转变就是，跟别人的沟通极大的增多了，这当然不是说自己很反对这个沟通，我只是讨厌低效的沟通，而且社会中这种低效的沟通又占了大多数，这个对于专注工作是不利的，我比较反感。我喜欢独处，当一个人独处的时候，负责的对象就只有自己，自己给自己设立目标，完成目标，简单直接，过程之中，自己可以控制节奏。我相信自己以后应该也能找到一份体面的工作，当基本生存忧虑消失后，我应该更加专注于自己，变化，成长，成为一个更好，更strong的人。

梦想

进入工作后，就很少思考这两个字了，以前在学校，被告知以专业知识为重，对于其他的东西，也都只有非常抽象的概念，梦想亦如此，校园里听着社会上很多前辈虔诚神圣的谈这个的时候，感觉我们几个，就像几岁的小孩，懵懵懂懂，只知这个重要，并非理解其中的意义，毕竟，在校园的日子里，是幸福的，过着衣食无忧的日子，还有一起简单学习的小伙伴，不懂了还有学院友善的师长，在这些简单单纯而未经过社会考验的我们，是断然不能理解这个东西的，关于这些的大道理也只是听听而已，真的就只是听听而已。 已经工作一年了，每周5个工作日，2个休息日，上班，朝九晚九，周末，就一个人宅在家里，有时就点点外卖，有时就一个人出去走走，每个月比较开心的日子就是发工资的前几天，看着银行账户上的几个数字在增长，私自为着自己能看到的一点点小努力而庆幸。日复一日，月复一月，有时感觉自己这样混日子，就像是在大大都市中的一个蝼蚁一般卑微，少了你，城市照样运转，多了你，城市也没什么变化，我突然想到我待在这里的意义是什么？我为什么要待在这里呢，当个体从群体中逐渐剥离后，个体内心对于这个社会的建构又好了吗？？？ 我在思索中。 也许我还没准备好 有时候，周末，一个人待在不到十平米的租房里发呆，我想回家，待在父母亲身边，多陪陪爷爷奶奶，操劳一生，自己又不想有‘子欲孝而亲不待’的遗憾，可是小城市上又能给我们这些年轻人什么呢，虽说安稳，但是谁又能保证，呆了几年之后，自己再出去，又会像以前出去念书一样，发出’’不知有汉，魏晋‘’的感叹呢，这种矛盾纠结，也许都需要我们这代这类人去完成，也许好一点的方案就是，在外努力拼搏，挣多点钱，在武汉买套房子，把父母接到城市中来，这样每日下班能见到自己的亲人，生活又会有动力了。 人的行为有很多种驱动点，有的人是利益，金钱能让他每天充满干劲，有的人是兴趣，对于专业的爱好足以让他在自己的世界里活的有滋有味，而有的人则是旁边人，亲人，爱人，他们的存在可以持续的让自己找到生活的动力，以前我以为我的驱动点是第一个，因为它能让我远离贫穷，远离拮据，远离穷酸的生活，上了大学之后我发现我的驱动点变成了第二个，专业知识的领域里，又让我找到一个无穷尽的世界，这里的逻辑，数学，吸引了我，当初的自己也以为这个会跟随着我的之后人生，但是进了社会，当专业的探索，处处都要与商业目标放在一起时，这里的兴趣，也逐渐的变了味，不可否认，技术是应当为人类服务的，但是强生生的把一个人的兴趣变化为商业化的东西时，这个就已经失去了兴趣的含义，进入工作以后，在夜半时分，在几十个周末一个人的日子里，脑子中的潜意识就会莫名其妙的想到过去，想到童年，仿佛，这就像十几年前时在脑海中种小的一颗种子一样，一直都未生根发芽，只不过在进入社会后，当环境，思想变荡巨大时，这粒种子突然茁壮得很快，直至变成一颗大树，存放在了脑海的主干位置，也许，我也正在变成一个由第三个来驱动的人了。

技术感悟-关于学习哪门子技术

上周跟领导谈了一次话，收益匪浅，摘录几点，总结一下。 现实中，只有极少数人的生存动力是真正的来源于兴趣，并还可以做出大贡献的，是那些极有天赋的人，像牛顿，爱因斯坦等，年少就已成才，这样的人，我们大多恐赶不及，而像我们一样的普通人绝大部分都来自于利益驱动，无论是哪一项技术，只要是能恰到好处的满足我们的需求，我们就不应因为兴趣的局限而逃避它，这当然不是说，开发人员时刻都要学会用时髦的it技术，这样的理解太狭隘，确定的了这一点，我们之后的一些就很明确了，既然利益是排在第一位的，那我们到底学习哪一门语言就显得不是那么重要了。 大学时候的常在想，自己就想只凭一项技术吃饭，把android学好就行了，现在想想，恐怕就太幼稚了，尤其是对这些已经很封装好了的的技术，(隐藏了底层的很多技术细节)，因此我们大多数人做的都是在别人基础上的工作，使用的是高级语言，就拿前端而言，前几年对前端的要求是，精通html、css、javaScript等，但在现在，仅凭这几门技术恐怕是很难找到一个比较好的工作的，因为新出来的框架，vue，angular，react等等设计，无论是在用户体验和开发体验，这些都比之前那些原生开发好太多了，所以如果想在这个编程行业能长期立足，我们就不能停止学习的脚步，只有在接触更多的东西以后，我们才会在对待新的东西变得游刃有余。是否能快速接受新事物并使用，也是评价我们核心竞争力的一个指标。 具体而言，衡量我们程序员之间的竞争力，一般包括这几个方面，专业的基础功底(第一重要)，编程的逻辑思维能力（思路是否清晰？是否考虑临界情况？异常处理等），编写的代码质量（即健壮性，可扩展性，可重用性），代码习惯（命名规范，函数使用等）等等这种能力，我们经常说一个人专不专业，大部分可以看这些。即使我们作为开发人员，能对这些要求烂熟于心，但在实际情况中，我们往往也达不到自己理想的标准，这与很多因素有关，需求背景，个人毅力，团队氛围等等。就像孟岩在他博客里说到的一样，我们国内的技术程序员，虽然代码功能都ok，但是就编程的这种基本素养，专业素质是无法与美国比较的，我们这个行业的程序员，很多是非常欠缺这种能力的，大多数的程序员都懒惰于执着于一门语言，不肯学新东西，因为他觉得老的技术还不是照样完成了功能，其实不光是程序员，也包括一些吝啬的老板，除非是在代码实在是维护不下去的时候，他才会考虑我们是不是该花点时间重构一下。有点偏题，回到正路。上面讲的这种能力就是一种非常重要的能力，如果我们能够通过日常的编码有意的训练这方面的能力，我想无论以后我们是选择哪门技术，哪门语言，我们都能在个人的范围内写出高质量的代码。 对于现在新出的技术，一般都是为了为了方便开发人员打造的，新技术不难学习，只是要花点时间，但是我们上面讲的，我们则可能要花几年去理解。 所以不用太纠结于学习哪一门技术。

github的一些操作

1.本地建立git仓库，先将窗口切换到本地项目下

git init

2.将项目的所有文件添加到仓库中

git add .

3.将add了的文件添加到本地仓库

git commit -m  ‘测试语句’

4.到github上创建自己的远程仓库repository,拿到创建仓库的https地址 5.将自己本地仓库关联到远程仓库中

git remote add origin  https://github.com/yf/Test1

6.传之前先pull下master分支

git pull  origin master

7.上传本地代码

git push -u  origin master
